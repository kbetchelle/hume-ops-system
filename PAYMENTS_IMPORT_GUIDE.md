# Arketa Payments Import Guide

## Quick Start

### 1. Upload Your CSV
1. Go to Backfill Manager
2. Click "Advanced Import with Mapping"
3. Select your `payments_1_of_2.csv` file
4. Choose table: **Arketa Payments**

### 2. Verify Auto-Mappings
The tool will automatically:
- Select `payment_id` as your unique key (from CSV)
- Map it to `external_id` in the database
- Map other columns based on matching names

### 3. Import
- Enable "Overwrite existing records" to update existing payments
- Click "Start Import"
- Review any skipped records in the detailed errors section

## Common Issues

### All Records Skipped
**Cause**: Missing `payment_id` values or schema mismatch
**Solution**: 
1. Check CSV has `payment_id` column with values in every row
2. Verify the unique key is auto-selected to `payment_id`
3. Check error details for specific row issues

### "Column does not exist" Error
**Cause**: Database schema doesn't have all CSV columns
**Solution**: Already fixed! The migration added all necessary columns.

### Duplicate Key Violations
**Cause**: Multiple rows with same `payment_id`
**Solution**: Enable "Overwrite existing records" to update duplicates instead of skipping

## Verify Import

After import, check your data:

```sql
-- Count total payments
SELECT COUNT(*) FROM arketa_payments;

-- View recent payments
SELECT 
  external_id,
  client_id,
  amount,
  status,
  payment_type,
  payment_date
FROM arketa_payments 
ORDER BY payment_date DESC 
LIMIT 10;

-- Check payment totals by status
SELECT 
  status,
  COUNT(*) as count,
  SUM(amount) as total_amount
FROM arketa_payments
GROUP BY status;

-- View refunded payments
SELECT 
  external_id,
  amount,
  amount_refunded,
  description,
  payment_date
FROM arketa_payments
WHERE amount_refunded > 0
ORDER BY payment_date DESC;
```

## Schema Details

The `arketa_payments` table includes:
- **Unique constraint** on `external_id` for deduplication
- **Indexes** on `client_id`, `payment_date`, `external_id`, `source`, `status`, and `location_name` for fast queries
- **RLS (Row Level Security)** enabled with policies for authenticated users
- **JSONB `raw_data`** column for storing complete API response if needed

## Tips

- **Batch Imports**: For large CSVs (>1000 rows), the tool automatically chunks into batches
- **Error Recovery**: Failed batches don't affect successful ones
- **Detailed Errors**: Click on any error to see the exact row data and issue
- **Updates vs Inserts**: With overwrite enabled, existing payments (same `external_id`) are updated with new data
